---------------------------------------------------------
  HOW IT WORKS
---------------------------------------------------------

devref stores your personal help sheets in two JSON files:

  S:\devref\ref\tools.json       Topics, use cases, descriptions
  S:\devref\ref\snippets.json    Snippet cards (patterns + examples)

Everything is searchable by topic, use case, example, or tag.

---------------------------------------------------------
  QUICK COMMAND REFERENCE
---------------------------------------------------------

devref --help                            Show all commands
devref --list                            List all tools

FINDING
  devref --find python                   Overview + all topics
  devref --find python --topic venv      Full detail on venv
  devref --find python --snippets        All python snippet cards
  devref --find python --snippets f-string  One snippet card

SEARCHING
  devref --search "environment"          Search across everything
  devref --search "environment" --tools     Search tools.json only
  devref --search "environment" --snippets  Search snippets.json only

  devref --find python --search "env"           Inside python, everything
  devref --find python --search "env" --topics     Topics only
  devref --find python --search "env" --snippets   Snippets only
  devref --find python --search "env" --usecases   Use cases only
  devref --find python --search "env" --examples   Examples only

ADDING
  devref --new git                       New tool wizard (terminal)
  devref --new git --notepad             New tool template in Notepad
  devref --add git --topic               Add topic to existing tool
  devref --add git --snippets            Add snippet to tool
  devref --add git --topic --notepad     Topic template in Notepad

EDITING
  devref --edit git --topic commit       Edit a topic (type name to confirm)
  devref --edit git --snippets stash     Edit a snippet

DELETING
  devref --delete git --topic commit     Delete one topic
  devref --delete git --snippets stash   Delete one snippet
  devref --delete git                    Delete entire tool

TAGS
  devref --tag python venv "beginner"    Tag a topic
  devref --find --tag "beginner"         Find all tagged entries

SETTINGS
  devref --set hints on                  Show example hints in wizard
  devref --set hints off                 Hide example hints in wizard

OTHER
  devref --recent                        Last 10 lookups
  devref --backup                        Backup files with timestamp
  devref --export python                 Export python as Markdown
  devref --import myfile.json            Merge external JSON into tools

AI PROMPT
  devref --prompt conda                  Print prompts for both files
  devref --prompt conda --tools          Prompt for tools.json only
  devref --prompt conda --snippets       Prompt for snippets.json only

---------------------------------------------------------
  JSON FILE STRUCTURES
---------------------------------------------------------

tools.json
----------
{
  "tool-name": {
    "description": "What this tool is",
    "tags": ["tag1", "tag2"],
    "use_cases": [
      "When you need to do X",
      "As an alternative to Y when Z"
    ],
    "topics": {
      "topic-name": {
        "description": "What this topic is about",
        "what_it_does": "Detailed explanation of behavior",
        "use_cases": [
          "Use this when doing X",
          "Prefer over Y for Z reasons"
        ],
        "syntax": [
          "command --flag <required>",
          "command --flag [optional]"
        ],
        "examples": [
          "concrete working example",
          "another real example"
        ],
        "tags": ["optional", "tags"]
      }
    }
  }
}

snippets.json
-------------
{
  "tool-name": {
    "entries": {
      "snippet-name": {
        "description": "What this pattern does",
        "use_cases": [
          "Use when building X",
          "Prefer over Y when Z"
        ],
        "pattern": "command --flag <required> [optional]",
        "examples": [
          "real example here",
          "another concrete example"
        ]
      }
    }
  }
}

---------------------------------------------------------
  USING AI TO BULK-GENERATE HELP SHEETS
---------------------------------------------------------

The fastest way to add a whole tool at once:

STEP 1  Run the prompt generator for the tool:
  devref --prompt conda --tools
  devref --prompt conda --snippets

STEP 2  Copy the printed prompt (one at a time)

STEP 3  Paste into Claude (claude.ai) or ChatGPT

STEP 4  Copy the returned JSON block

STEP 5  Save it as a .json file (e.g. conda_tools.json)

STEP 6  Import it:
  devref --import conda_tools.json

Run the same for snippets and import that too.

---------------------------------------------------------
  WIZARD HINTS
---------------------------------------------------------

When adding entries via the wizard, devref shows grey example
hints above each prompt so you know what format to use.

To toggle hints:
  devref --set hints on
  devref --set hints off

---------------------------------------------------------
  INTERACTIVE AUTOCOMPLETE
---------------------------------------------------------

devref uses prompt_toolkit for smart autocomplete in the wizard.
When prompted for a topic name or tool name, start typing and
press Tab or use arrow keys to navigate suggestions.

This works in Windows Terminal, PowerShell, and CMD.

---------------------------------------------------------
  FILE LOCATIONS
---------------------------------------------------------

  S:\devref\devref.exe            The CLI tool
  S:\devref\ref\tools.json        Your topics and help entries
  S:\devref\ref\snippets.json     Your snippet cards
  S:\devref\ref\meta.json         Settings + recent history
  S:\devref\backups\              Timestamped backups
  S:\devref\devref_guide.txt      This file

---------------------------------------------------------
  TIPS
---------------------------------------------------------

  - Run devref --backup before making large edits
  - The --export command creates a .md file you can open
    in Obsidian, Notion, or any Markdown viewer
  - JSON files are plain text -- edit directly in any
    editor if you prefer (keep valid JSON format)
  - If you break a JSON file, restore from S:\devref\backups\
  - Use tags to filter across tools (e.g. tag everything "beginner")
  - devref --prompt generates separate prompts for tools and snippets
    to prevent AI from getting confused by a combined structure
