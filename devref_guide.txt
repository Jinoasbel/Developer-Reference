  HOW IT WORKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

devref stores your personal help sheets in two JSON files:

  S:\devref\ref\ref.json       Topics, descriptions, examples
  S:\devref\ref\syntax.json    Syntax cards (shorthand patterns)

You can add any tool, language, or workflow to these files.
Everything is searchable, tagged, and retrievable from any terminal.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  QUICK COMMAND REFERENCE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

devref --help                            Show all commands
devref --list                            List all tools

FINDING
  devref --find python                   Overview of python + all topics
  devref --find python --topic venv      Full detail on venv topic
  devref --find python --syntax          All python syntax cards
  devref --find python --syntax f-string One syntax card

SEARCHING
  devref --search "environment"          Search everywhere
  devref --find python --search "env"    Search only inside python

ADDING
  devref --new git                       New tool wizard (terminal)
  devref --new git --notepad             New tool template in Notepad
  devref --add git --topic               Add topic to existing tool
  devref --add git --syntax              Add syntax entry to tool
  devref --add git --topic --notepad     Topic template in Notepad

EDITING
  devref --edit git --topic commit       Edit a topic (type name to confirm)
  devref --edit git --syntax stash       Edit a syntax entry

DELETING
  devref --delete git --topic commit     Delete one topic
  devref --delete git                    Delete entire tool entry

TAGS
  devref --tag python venv "beginner"    Tag a topic
  devref --find --tag "beginner"         Find all beginner-tagged entries

OTHER
  devref --recent                        Last 10 lookups
  devref --backup                        Backup JSON files with timestamp
  devref --export python                 Export python as Markdown file
  devref --import myfile.json            Merge external JSON into ref

AI PROMPT HELPER
  devref --prompt conda                  Print the AI prompt for conda

  JSON FILE STRUCTURE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ref.json structure:
─────────────────
{
  "tool-name": {
    "description": "What this tool is",
    "tags": ["tag1", "tag2"],
    "topics": {
      "topic-name": {
        "description": "What this topic is about",
        "what_it_does": "Detailed explanation of behavior",
        "syntax": [
          "command --flag <arg>",
          "command --other-flag"
        ],
        "examples": [
          "concrete working example",
          "another real example"
        ],
        "tags": ["optional", "tags"]
      }
    }
  }
}

syntax.json structure:
──────────────────────
{
  "tool-name": {
    "entries": {
      "syntax-name": {
        "description": "What this syntax pattern does",
        "pattern": "command --flag <required> [optional]",
        "examples": [
          "real example here"
        ]
      }
    }
  }
}

  USING AI TO BULK-GENERATE HELP SHEETS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

The fastest way to add a whole tool at once:

STEP 1 — Run the prompt generator:
  devref --prompt conda

STEP 2 — Copy the printed prompt

STEP 3 — Paste it into Claude (claude.ai) or ChatGPT

STEP 4 — The AI will return a JSON block. Copy it.

STEP 5 — Paste it into a .json file, e.g. conda.json
  (Save it somewhere accessible, like your Desktop)

STEP 6 — Import it:
  devref --import C:\Users\YourName\Desktop\conda.json

That's it. The tool is now in your reference.

• Run  devref --backup  before making large edits
• Tags let you filter across tools (e.g. tag everything "beginner")
• The --export command creates a .md file you can open in Obsidian,
  Notion, or any Markdown editor
• JSON files are plain text — you can edit them directly in any
  text editor if you prefer (just keep valid JSON format)
• If you break a JSON file, restore from a backup in S:\devref\backups\
